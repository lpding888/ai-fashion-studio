V1.0

## 一、快速开始

### 1.1 系统要求

**开发环境（本地开发）**:
- CPU: 双核及以上
- 内存: 8GB 及以上
- 存储: 20GB 可用空间
- 操作系统: macOS 10.15+ / Windows 10+ / Linux (Ubuntu 18.04+)
- Node.js: v18.17.0 或更高版本
- pnpm: v8.x 或更高版本
- Docker: 20.10.0 或更高版本
- 浏览器: Chrome 90+ / Firefox 88+ / Safari 14+ / Edge 90+

**生产环境（服务器部署）**:
- CPU: 2核及以上
- 内存: 4GB 及以上
- 存储: 50GB 可用空间
- 操作系统: Linux (Ubuntu 18.04+ / CentOS 7+)
- Node.js: v18.17.0 或更高版本
- Docker: 20.10.0 或更高版本
- Nginx: 1.18+ （推荐）
- PM2: 5.0+ （推荐）

### 1.2 开发环境安装

#### 步骤1: 安装Node.js和pnpm

**macOS/Linux**:
```bash
# 安装Node.js (使用nvm)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18.17.0
nvm use 18.17.0

# 安装pnpm
npm install -g pnpm
```

**Windows**:
1. 访问 https://nodejs.org/ 下载Node.js 18.17.0 LTS版本并安装
2. 打开PowerShell运行: `npm install -g pnpm`

#### 步骤2: 安装Docker

**macOS**:
1. 下载Docker Desktop for Mac: https://www.docker.com/products/docker-desktop/
2. 安装并启动Docker Desktop

**Windows**:
1. 下载Docker Desktop for Windows: https://www.docker.com/products/docker-desktop/
2. 启用WSL2并安装Docker Desktop

**Linux**:
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
```

#### 步骤3: 克隆项目

```bash
git clone <repository-url>
cd ai-fashion-studio
```

#### 步骤4: 安装依赖

```bash
# 安装后端依赖
cd server
pnpm install

# 安装前端依赖
cd ../client
pnpm install
```

#### 步骤5: 配置环境变量

**后端配置** (server/.env):
```env
# 数据库配置
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/fashion_studio"

# JWT密钥
JWT_SECRET="your-jwt-secret-key"

# Gemini API配置
GEMINI_API_KEY="your-gemini-api-key"

# 腾讯云配置
TENCENT_COS_SECRET_ID="your-secret-id"
TENCENT_COS_SECRET_KEY="your-secret-key"
TENCENT_COS_BUCKET="your-bucket-name"
TENCENT_COS_REGION="ap-guangzhou"

# 翻译API配置（可选）
TRANSLATION_API_KEY="your-translation-api-key"
```

**前端配置** (client/.env.local):
```env
NEXT_PUBLIC_API_URL=http://localhost:3000
```

#### 步骤6: 初始化数据库

```bash
# 启动PostgreSQL数据库
docker-compose up -d

# 运行数据库迁移
cd server
pnpm prisma migrate dev
```

### 1.3 启动应用

#### 方式一: 一键启动（推荐）

**macOS/Linux**:
```bash
./dev.sh
```

或直接双击运行 `启动开发环境.command` (macOS)

**Windows**:
```bash
.\start.bat
```

#### 方式二: 手动启动

**启动数据库**:
```bash
docker-compose up -d
```

**启动后端**:
```bash
cd server
pnpm run start:dev
# 后端将运行在 http://localhost:3000
```

**启动前端**:
```bash
cd client
pnpm run dev
# 前端将运行在 http://localhost:3001
```

### 2.2 任务管理

#### 2.2.1 创建任务

**方式一: 传统工作流**

1. 点击首页的"创建任务"按钮
2. 上传服装图片（1-20张）
   - 支持JPG、PNG格式
   - 单张图片不超过10MB
   - 可拖拽上传或点击选择
3. （可选）上传人脸参考图
4. 填写需求表单：
   - **用途/平台**: 选择图片使用平台（小红书、淘宝、Instagram等）
   - **风格关键词**: 输入风格描述（如：潮牌、高街、休闲等）
   - **统一大场景**: 描述整体拍摄场景（如：街头、工作室、户外等）
   - **张数**: 设置生成图片数量（1-20张）
   - **动作偏好**: （可选）选择人物动作类型
   - **约束条件**: （可选）设置图片约束（如：不出现logo、背景干净等）
5. 点击"生成计划"按钮
6. AI自动生成Shot List和GARMENT LOCK
7. 检查并编辑生成计划：
   - 查看GARMENT LOCK（服装关键特征）
   - 查看Shot List（每张图的拍摄计划）
   - 查看每张图的英文Prompt
   - （可选）编辑GARMENT LOCK
   - （可选）编辑Shot List
   - （可选）编辑Prompt
8. 点击"确认并开始生成"
9. 系统逐张生成图片
10. 实时查看生成进度
11. 生成完成后，进入图片质检环节

**方式二: 直出工作流**

1. 点击首页的"直出创建"按钮
2. 上传参考图片
3. 输入直接Prompt（英文）
4. 配置生成参数：
   - **分辨率**: 选择图片分辨率（512x512、768x768、1024x1024等）
   - **比例**: 选择图片比例（1:1、3:4、4:3、16:9等）
   - **模型选择**: 选择使用的AI模型
   - **张数**: 设置生成数量
5. 选择生成模式：
   - **Grid模式**: 网格形式批量生成
   - **Individual模式**: 单独生成每张图
6. 点击"开始生成"
7. 查看生成结果
8. （可选）下载或重新生成

#### 2.2.2 查看任务历史

1. 点击页面顶部的"历史"标签
2. 查看任务列表：
   - 任务名称
   - 创建时间
   - 生成状态
   - 图片数量
   - 完成进度
3. 使用筛选器过滤任务：
   - 按状态筛选（进行中、已完成、失败）
   - 按时间范围筛选
4. 点击任务卡片查看详情

#### 2.2.3 查看任务详情

1. 在任务列表中点击任务卡片
2. 查看任务完整信息：
   - 任务基本信息（名称、时间、状态）
   - 上传的参考图片
   - 填写的需求参数
   - 生成的Shot List和Prompt
   - 生成结果图片
   - 修复历史记录
3. 查看每张图片的详细信息：
   - 生成时间
   - 使用的Prompt
   - 生成参数
   - 修复记录
4. 操作按钮：
   - **下载**: 下载单张图片
   - **编辑**: 在线编辑图片
   - **修复**: 进入修复流程
   - **删除**: 删除该图片

#### 2.2.4 删除任务

1. 进入任务详情页
2. 点击右上角的"删除任务"按钮
3. 确认删除操作

### 2.3 图片生成和质检

#### 2.3.1 图片生成

任务创建后，系统会自动开始生成图片：

1. **逐张生成模式**:
   - 按Shot List逐张生成
   - 每张图生成完成后自动进入质检
   - 可随时暂停和继续

2. **批量生成模式**:
   - 多张图并行生成
   - 生成完成后统一质检

3. **查看生成进度**:
   - 实时显示生成进度条
   - 显示当前正在生成的图片
   - 显示已完成的图片数量

#### 2.3.2 图片质检

每张图片生成完成后，需要进行质检：

1. **查看生成结果**:
   - 大图预览
   - 与参考图对比
   - 查看使用的Prompt

2. **质检选项**:
   - **通过**: 图片质量满意，继续下一张
   - **需要修复**: 图片有瑕疵，进入修复流程
   - **重新生成**: 不满意，重新生成该张

3. **批量质检**:
   - 可以批量标记多张图片
   - 批量执行通过、修复、重新生成操作

#### 2.3.3 图片修复

选择"需要修复"后，进入智能修复流程：

1. **差异对比**:
   - AI自动分析生成图与参考图的差异
   - 生成差异清单（哪些地方不一致）

2. **修复方案生成**:
   - AI生成修复建议
   - 生成修复指令草稿
   - 用户可以编辑修复指令

3. **执行修复**:
   - 确认修复方案
   - 点击"执行修复"
   - 系统仅修复衣服细节
   - 保持模特、场景、光线等不变

4. **修复结果质检**:
   - 查看修复后图片
   - 满意则通过，不满意可以继续修复或重新生成

> 💡 提示：修复原则是只改衣服细节，模特脸、姿势、场景、光线、镜头都不变。

## 三、常见问题解答

### 3.1 安装和启动问题

**Q1: 启动时提示端口被占用怎么办？**

A: 可以修改默认端口：
- 后端端口: 修改 `server/src/main.ts` 中的 `3000`
- 前端端口: 修改 `client/next.config.ts` 中的 `3001`
- 数据库端口: 修改 `docker-compose.yml` 中的 `5432`

**Q2: 数据库连接失败怎么办？**

A: 请检查：
1. Docker是否正在运行：`docker ps`
2. PostgreSQL容器是否启动：`docker-compose ps`
3. 数据库连接字符串是否正确：`DATABASE_URL`环境变量
4. 防火墙是否阻止了连接

**Q3: npm install失败怎么办？**

A: 尝试以下方法：
1. 清除npm缓存：`npm cache clean --force`
2. 删除node_modules和lock文件：
   ```bash
   rm -rf node_modules
   rm -f package-lock.json
   npm install
   ```
3. 使用国内镜像源：
   ```bash
   npm config set registry https://registry.npmmirror.com
   ```

**Q4: pnpm安装失败怎么办？**

A: 尝试以下方法：
1. 使用npm全局安装：`npm install -g pnpm`
2. 使用yarn替代pnpm（需要修改package.json）
3. 使用curl安装（Linux/macOS）：
   ```bash
   curl -fsSL https://get.pnpm.io/install.sh | sh -
   ```

### 3.2 功能使用问题

**Q5: 图片生成失败怎么办？**

A: 请检查：
1. API Key是否正确配置
2. API额度是否充足
3. 网络连接是否正常
4. 参考图片格式是否正确（支持JPG、PNG）
5. 参考图片大小是否超过限制（单张10MB）

**Q6: 图片生成很慢怎么办？**

A: 图片生成速度主要取决于：
1. AI API的响应时间（通常10-30秒/张）
2. 网络延迟
3. 服务器性能

优化建议：
1. 使用批量生成模式提高效率
2. 避免在高峰时段生成
3. 检查网络连接质量

**Q7: 修复效果不理想怎么办？**

A: 修复效果受以下因素影响：
1. 修复内容的复杂程度
2. 参考图的清晰度
3. 修复指令的准确性

优化建议：
1. 编辑修复指令，更精确地描述需要修改的地方
2. 使用高清晰度的参考图
3. 如果修复多次仍不满意，建议重新生成

**Q8: 如何提高生成质量？**

A: 提高生成质量的建议：
1. 上传高质量的参考图片
2. 详细描述需求参数
3. 合理使用预设系统
4. 适当调整GARMENT LOCK
5. 参考优秀案例的Prompt

**Q9: 积分不足怎么办？**

A: 解决方法：
1. 在个人中心充值积分
2. 邀请新用户注册获取奖励积分
3. 联系管理员申请积分

**Q10: 任务无法删除怎么办？**

A: 请检查：
1. 任务是否正在进行（进行中的任务无法删除）
2. 是否有足够的权限（普通用户只能删除自己的任务）
3. 如果以上都正常，可能是系统bug，请联系管理员

### 3.3 性能和优化问题

**Q11: 页面加载很慢怎么办？**

A: 优化建议：
1. 清除浏览器缓存
2. 检查网络连接
3. 关闭占用带宽的其他程序
4. 使用Chrome等高性能浏览器

**Q12: 前端构建很慢怎么办？**

A: 优化建议：
1. 使用增量构建：`pnpm build`
2. 清除构建缓存：删除 `.next` 目录后重新构建
3. 增加系统内存
4. 使用SSD硬盘

**Q13: 数据库查询很慢怎么办？**

A: 优化建议：
1. 检查数据库索引
2. 使用Prisma的查询优化功能
3. 增加数据库内存配置
4. 考虑使用数据库缓存（Redis）

**Q14: 如何备份和恢复数据？**

A: 备份和恢复步骤：

**备份数据库**:
```bash
# 备份数据库到文件
docker exec -t postgres_container pg_dump -U postgres fashion_studio > backup.sql
```

**恢复数据库**:
```bash
# 从备份文件恢复数据库
cat backup.sql | docker exec -i postgres_container psql -U postgres fashion_studio
```

**备份图片文件**:
- 腾讯云COS自动提供备份功能
- 可以定期导出重要图片到本地

3. SQL注入防护
4. XSS和CSRF防护

3. 定期更换API Key
4. 不要在代码中硬编码API Key

**Q18: 如何保护账号安全？**

A: 建议：

2. 不要使用生日、手机号等容易猜到的密码
3. 定期更换密码
4. 不要在公共电脑上登录
5. 登录后及时退出
6. 开启邮箱验证功能

### 4.3 术语表

**术语解释**:
- **Brain**: AI策划模块，负责需求分析和计划生成
- **Painter**: AI执行模块，负责图像生成
- **GARMENT LOCK**: 服装关键特征锁定，确保多图一致性
- **Shot List**: 拍摄计划列表，包含每张图的拍摄指令
- **Prompt**: AI生成的指令，用于指导图像生成
- **预设**: 可复用的参数配置（风格、人脸、姿势）
- **质检**: 对生成图片进行质量检查和验收
- **修复**: 对不满意的图片进行精确修改
- **直出模式**: 快速生成模式，直接输入Prompt生成图片
