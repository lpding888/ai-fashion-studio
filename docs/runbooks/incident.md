## 事故处理 Runbook（本地 MVP）

### 0) 分级（建议）
- P1：核心流程不可用（生成计划/出图/导出大面积失败），或数据损坏风险
- P2：部分功能异常/可绕过（单个页面/非核心导出异常）

### 1) 立即止血
- [ ] 暂停高风险操作（导出/批量生成/写入）
- [ ] 如是最近改动导致：回滚到上一个稳定版本

### 2) 快速定位
- [ ] 通过 request_id 找到一次失败请求日志
- [ ] 判断阶段：Brain / Painter / DB / 文件系统 / UI
- [ ] 记录外部 API 的状态码与错误摘要（不要把 key 打到日志里）

### 3) 恢复服务
- [ ] 修复/回滚完成
- [ ] P0 冒烟通过（见 `docs/runbooks/release.md`）

### 4) 复盘与行动项
- [ ] 建立 postmortem：`docs/runbooks/postmortem_template.md`
- [ ] 行动项进入 backlog（owner + 截止日期 + 验证方式）
