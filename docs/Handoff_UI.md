# UI 交接包（给设计师/走 Figma）

## 交付物清单（按优先级）
- `docs/PM_Brief.md`：业务目标 + P0 范围冻结 + 非目标
- `docs/UX_Spec.md`：页面/流程/状态/文案（可直接画原型）
- `docs/Contracts.md`：字段契约（表格列、状态、修复输出结构）
- `docs/Concept_Dictionary.md`：术语对齐（避免“同词不同义”）
- `docs/UI_Wireframes.md`：低保真线框（页面布局/组件位）
- `docs/UX_Spec.json`：结构化 UX_Spec（便于后续开发对齐）

## 需要 UI 重点解决的 4 个体验问题
1) **参考图 vs 输出图的对比效率**：并排、放大、快速切换 Shot（键盘/快捷操作可后置）
2) **GARMENT LOCK 的编辑体验**：像“标签/清单”一样轻量，可排序；支持“全局锁定点 + 本行覆盖”
3) **逐张队列出图的可控性**：可见进度、单张失败可重试、不阻塞其他
4) **修复弹窗的两步确认**：先生成修复方案（可编辑）再执行修复（生成新版本、可回退）

## 默认定稿建议（如无异议按此走）
1) 任务详情主导航：**Stepper（可点击回到上一步）**  
   - 理由：流程天然是“计划→出图→质检→导出”，用 Stepper 降低理解成本；仍保留可回退编辑
2) 质检页布局：**左列表右大图（折中）**  
   - 理由：需要“快速切 Shot + 看大图细节”，该布局信息密度与可用性最平衡
3) GARMENT LOCK 呈现：**Tag 展示 + 编辑弹窗（组合）**  
   - 理由：日常浏览用 Tag 一眼扫过；编辑时用弹窗做增删改排序更顺手

## 页面列表（P0）
- Home：任务列表（新建/设置入口）
- New Task：上传 + 需求表单 + 生成计划
- Task Detail（Stepper）：计划 → 出图 → 质检 → 导出
- Fix Modal：修复弹窗（对比 + 生成修复方案 + 执行修复）
- Settings：Base URL / API Key / 默认输出 / 测试连接

## UI 需要你做的关键决定（选择题，便于定稿）
1) 任务详情的主导航：A Tab（计划/出图/质检/导出）B Stepper（1计划→2出图→3质检→4导出）
2) 质检页信息密度：A 卡片瀑布流（大图优先）B 表格（效率优先）C 左列表右大图（折中）
3) GARMENT LOCK 呈现：A 可排序 Tag 列表 B Checkbox 清单 C 两者组合（Tag 展示 + 编辑弹窗）

## 视觉方向（建议，不强制）
- 工具型 Admin：克制、信息密度高、状态明确
- 主色中性 + 强调色用于状态（成功/失败/进行中）
- 图像预览区域要“够大”，避免反复点放大
